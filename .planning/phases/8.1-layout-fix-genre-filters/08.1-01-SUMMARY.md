# Phase 8.1 Plan 01: Layout Fix + Genre Filters Summary

## One-liner

Genre filters fetched from Jikan API displayed in right sidebar panel (lg+); left-aligned anime cards; mobile shows floating Filters button

## Implementation

### Task 1: Add genre API function
- Added `getGenres()` function to fetch from Jikan `/genres/anime` endpoint
- Added `Genre` interface with `mal_id` and `name`
- Added fallback genre list (43 genres) if API fails
- Added `genre` parameter support to `getTopAnime`, `searchAnime`, `getLatestAnime`, `getSeasonalAnime`

### Task 2: Add genre state to useSearch hook
- Added `genres: Genre[]` state loaded on mount
- Added `selectedGenreId: number | null` for single-select
- Added `setGenre` function to set/clear genre filter
- Modified all API calls to pass genre parameter

### Task 3: Restructure SearchResults with right panel
- Container: flex layout with main content + right panel
- Main content: grid with `justify-start` (left-aligned), `grid-cols-2 md:grid-cols-3 lg:grid-cols-4 xl:grid-cols-5`
- Right panel: 260px fixed width, `hidden lg:block`, sticky positioning at top-4
- Mobile: floating "Filters" button, expandable drawer with genre badges, category, and view selector

### Task 4: Simplify SearchBar
- Removed category filter badges (moved to right panel)
- Removed default view selector (moved to right panel)
- Kept: search input, title language toggle, clear button, error display
