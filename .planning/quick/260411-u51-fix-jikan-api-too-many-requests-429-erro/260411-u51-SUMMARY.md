# Quick Task 260411-u51: Fix Jikan API 429 Too Many Requests Errors

**One-liner:** Promise-chain rate limiter with 2-second 429 backoff, mount-guard dedup, and AbortSignal cancellation to eliminate concurrent-request races against Jikan's 3-req/sec limit.

## What Was Built

### Task 1 — Serialized rate-limit queue + 429 backoff (src/lib/api.ts)

**Root cause fixed:** The old `respectRateLimit()` was an eager read/write — concurrent callers all read `lastRequestTime=0` before any caller updated it, so multiple fetches fired simultaneously within the same ~0ms window.

**Changes:**
- Replaced `lastRequestTime` + eager `respectRateLimit` with a `rateLimitChain` promise queue. Each caller chains onto the tail of the queue before awaiting, so concurrent callers serialize automatically.
- Added `rateLimitCooldownUntil` module-level variable: when a 429 is received, all subsequent calls wait out the remaining cooldown before entering the queue.
- Extracted a `fetchJikan(url, externalSignal?)` helper that: waits for any active cooldown, awaits the rate-limit queue, sets a 10-second fetch timeout, retries once on 429 (with 2-second backoff), and returns a normalized `JikanResult`.
- Refactored `getTopAnime`, `searchAnime`, `getLatestAnime`, `getSeasonalAnime` to build their URL then delegate to `fetchJikan(url, signal)` — each function body reduced from ~50 lines to 4-5 lines.
- Added `signal?: AbortSignal` as last parameter to all four functions.
- `MIN_REQUEST_DELAY` bumped from 350ms to 400ms.
- `getGenres` left unchanged (one-time cached fetch, separate concern).

### Task 2 — Mount guard + AbortSignal wiring (src/hooks/useSearch.ts)

**Root cause fixed:** The filter-change `useEffect` fires on first render with its initial state values, triggering a second `loadTopAnime(1)` call within ~0ms of the mount-only effect — two simultaneous API calls on startup.

**Changes:**
- Added `filtersInitialized` ref: the filter-change effect sets it to `true` on first call and returns early, so only subsequent filter changes trigger `loadTopAnime`.
- Added `browseAbortControllerRef`: `loadTopAnime` aborts any in-flight browse request before starting a new one (mirrors the existing `abortControllerRef` pattern for search).
- `loadTopAnime` now passes `signal` to `getLatestAnime`, `getSeasonalAnime`, and `getTopAnime`.
- `searchAnime` call in the debounce effect now passes `abortControllerRef.current.signal` as the sixth argument.
